None

The dblhashmap should be faster (have less collisions) on average since it is more random.  The linear hash map is much more simple
it is just h+1%capacity.  This will mean that every time it hits a collision, it will go to the next value and the next.  It's more
likely to cluster.  However, the dblhashmap is much more random since it is h(k) + f(i) where i is the number of iterations.
